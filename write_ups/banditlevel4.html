Command Explanation
ls List directory contents
cd Change the working directory
cat Concatenate files and print on the standard output
file Determine file type
du Estimate file space usage
find Search for files in a directory hierarchy

Write up

  1. Step one would be to locate the inhere directory. Run the ls command to pull the list of files and folders. This shows that the inhere directory is in the home directory. Now to change the directory that we are in we need to use the ‘cd’ command (cd inhere). bandit4@bandit:~$ ls
    inhere
  2. Step two, Now that we are in the inhere directory we need to search for a human readable file. Now let's use the ‘ls’ command with the ‘-a’ flag (ls -a) just in case there is a hidden file. The ‘ls -a’ command shows that there are 10 files. At this point we want to go through each file one-by-one and find that one that is readable. bandit4@bandit:~/inhere$ ls -a
    .  -file00 -file02 -file04 -file06 -file08
    .. -file01 -file03 -file05 -file07 -file09
  3. Step three, we need to open or read the files, you will notice that there is a dash in front of each file. This is part of the file name, so in order for the command prompt not to be confused and take the dash as an argument you must give the full file path to open the file. Since we are in the directory of the file we simply need to type a ./ in front of the file name to denote that the file is in this directory. So we use the cat command on each file. bandit4@bandit:~/inhere$ cat ./-file07
    lrIWWI6bB37kxfiCQZqUdOIYfr6eEeqR     And -file07 has the flag that we wanted to capture. This is the password to log into bandit level 5. (lrIWWI6bB37kxfiCQZqUdOIYfr6eEeqR)
  4. And don’t forget to use the login username of bandit5 for the next level.

Alternate Solutions

  • An alternative way to look through the files and find the human readable one is to change the file type by using the file command plus the file name. In this scenario the unreadable files will be of type data and the readable file will be of type ASCII text. bandit4@bandit:~/inhere$ file ./-file06
    ./-file06: data
    bandit4@bandit:~/inhere$ file ./-file07
    ./-file07: ASCII text
  • And with the file command you can use an * for a wildcard to search all the files in a single command call. Don’t forget to put the ./ in front of the asterisk since the files start with a dash (file ./*) bandit4@bandit:~/inhere$ file ./*
    ./-file00: data
    ./-file01: data
    ./-file02: data
    ./-file03: data
    ./-file04: data
    ./-file05: data
    ./-file06: data
    ./-file07: ASCII text
    ./-file08: data
    ./-file09: data
